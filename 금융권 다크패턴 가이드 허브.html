📋 금융권 다크패턴 규제 개요
금융위원회가 2025년 12월 발행한 가이드라인입니다. 온라인 금융거래에서 소비자의 합리적 의사결정을 침해하는 모든 다크패턴을 규율합니다. 광고→계약→유지→해지 전 단계를 포함합니다.

⚠️ 왜 금융권인가?

금융상품은 무형의 지속적 거래로, 일반 전자상거래 대비 정보비대칭이 심합니다. 거래액이 크고 소비자 피해 규모가 훨씬 크기 때문에 특별한 보호가 필요합니다. 디지털에 취약한 소비자상을 전제로 설계되어야 합니다.

금융상품판매업자, 금융상품자문업자, 혁신금융서비스 지정 핀테크, 금융회사 대리/연계 온라인 플랫폼
📱
대상 인터페이스
웹사이트, 모바일 앱, 모든 온라인 인터페이스를 통한 금융상품 및 금융서비스 판매/권유

정보 수집에 과도한 시간·노력을 요구해 선택을 포기하도록 유도 (4개 유형)
고위험
⚡ 압박형
심리적 압박으로 특정 행위를 반복 요구 (5개 유형)
중위험
💰 편취유도형
알아채기 어려운 조작으로 비합리적 지출을 유도 (1개 유형)
고위험

📌 일반원칙

  • 금융소비자가 거래 조건을 명확하고 쉽게 이해할 수 있도록 표시
  • 제한된 화면에서 오인 없이 합리적 결정을 할 수 있도록 설계
  • 선택을 왜곡 또는 침해하는 다크패턴 금지
🎭 오도형 다크패턴 (5개 유형)
거짓 정보 또는 통상적 기대와 다른 화면·문장 구성으로 금융소비자의 착각·실수를 유도하는 행위
①
설명절차의 과도한 축약
확인 단계 생략으로 부주의한 의사결정 유도
금지 행위:
  • 확인 페이지 같은 단계 제거
  • 개별 이해확인 절차 생략
  • 상품 이해도 일괄 확인 처리
❌ 위반 사례: 투자성 상품 권유 시 각 상품별로 이해여부를 확인해야 하는데, 모든 상품에 대해 "이해했음"으로 일괄 표시하도록 하는 경우
금융소비자보호법 제19조 위반
②
속임수 질문
의도하지 않은 답변을 유도하는 복잡한 질문
금지 행위:
  • 2가지 이상을 한꺼번에 묻기
  • 모호한 표현으로 혼동 유발
  • 주의깊게 봐야만 정확히 알 수 있는 질문
❌ 위반 사례: "상품 신청 및 마케팅 정보 수신에 동의하십니까?"라고 묻기 (어떤 것에 동의하는지 불명확)
의도 모호성
③
잘못된 계층구조
시각적 강조로 소비자에게 불리한 선택 유도
금지 행위:
  • 사업자 유리 버튼을 크고 진하게 표시
  • 소비자 유리 버튼을 작고 흐리게 표시
  • 필수 선택이 아닌 항목을 필수인 것처럼 표시
❌ 위반 사례: "신청하기" 버튼은 검은색 크게, "다음에 하기" 버튼은 배경색과 같이 표시하거나, 동의 버튼은 크게 상단에, X 버튼은 작게 우측 모서리에 배치
시각적 편향
④
특정옵션의 사전선택
사업자 유리 옵션을 미리 선택해 놓기
금지 행위:
  • 마케팅 정보 수신 동의 미리 체크
  • 우대 상품이 기본값으로 설정
  • 추가 상품 일괄 선택 상태
❌ 위반 사례: 실손보험 가입 시 "함께 가입" 옵션이 이미 체크되어 있어서 소비자가 이를 지나치게 되거나 필수로 오인하는 경우
암묵적 동의
⑤
허위광고 및 기만적 유인
거짓 정보로 소비자를 속이는 행위
금지 행위:
  • 근거 없는 거짓/과장 광고
  • 혜택만 강조, 손실 정보 은폐
  • 수익률 보장하는 단정적 표현
  • 사실 일부 조작·누락·축소
❌ 위반 사례: "수익 보장" 표현 또는 조건 없이 최고 수익률만 광고하고 실제 달성 조건 숨기기, 최저금리만 표시하고 우대조건 미표시
표시광고법 위반

✅ 오도형 회피 방법

  • 설명·확인·이해 절차를 생략하지 않기
  • 질문은 하나의 명확한 뜻만 담기
  • 동의·비동의 버튼을 동일 크기·색상으로 표시
  • 선택사항은 모두 미리 선택 해제 상태로
  • 모든 수치·주장에 정확한 근거 제시
🚧 방해형 다크패턴 (4개 유형)
의사결정에 필요한 정보 수집·분석에 과도한 시간·노력·비용이 들게 만들어 합리적인 선택을 포기하도록 유도하는 행위
①
취소·탈퇴 등의 방해
해지가 가입보다 복잡한 절차
금지 행위:
  • 취소 버튼을 숨기거나 접근 경로 불명확
  • 가입보다 훨씬 복잡한 해지 절차
  • 통합앱 탈퇴와 서비스 탈퇴 혼동
  • 필수 안내사항 외 불필요한 단계 추가
❌ 위반 사례: 가입: 3클릭이지만 해지: 15클릭 필요 | 해지 버튼을 찾기 어려운 위치에 배치 | 고객센터 상담원 연결을 의무화하되 명확히 안내하지 않음
금융소비자보호법 위반
②
숨겨진 정보
중요 정보를 은폐·누락·축소
금지 행위:
  • 설명서 미다운로드 상태에서 진행 허용
  • "자세히 보기" 클릭 필요한 정보 숨기기
  • 좌우 스크롤 필요한 정보 배치
  • 손실 정보는 작고 흐리게 표시
❌ 위반 사례: 금융소비자의 권리, 수수료, 환매 제한 조건 등을 기본 화면에서 보이지 않게 숨기기
온라인 설명의무 위반
③
가격비교 방해
다른 상품과의 비교를 어렵게 만들기
금지 행위:
  • 우대 조건·자격 기준 숨기기
  • 한 상품은 최고 이율, 다른 상품은 최저 이율로 표시
  • 통일되지 않은 기준으로 비교 유도
  • 조건을 은폐·축소·누락
❌ 위반 사례: A대출(우대금리): 3.0% | B대출(평균 이율): 8.5% 표시 (실제 A도 최대 7%, B도 최소 2%일 수 있음)
부당권유 가능성
④
클릭 피로감 유발
원하는 선택지까지 너무 많은 클릭 필요
금지 행위:
  • 간단한 선택을 여러 메뉴 거쳐서 하도록
  • 불필요한 질문에 반복 응답 요구
  • 취소/환불 메뉴까지 과도한 경로
  • 전체선택·전체취소 기능 없음
❌ 위반 사례: 마이데이터 연결: 1클릭으로 가능하지만 마이데이터 해제: 5개 메뉴를 거쳐야 함
합리적 선택 방해

✅ 방해형 회피 방법

  • 가입과 동일 수준의 해지 절차 제공
  • 중요 정보를 화면에 직관적으로 노출
  • 모든 상품을 통일된 기준으로 비교 표시
  • 클릭 수 최소화 (1-2클릭 권장)
  • 복잡한 경우 전체선택/취소 버튼 제공
⚡ 압박형 다크패턴 (5개 유형)
금융소비자에게 심리적 압박을 가해 특정 행위를 반복 요구하거나 감정적·감각적으로 조작하는 행위
①
계약과정 중 기습적 광고
거래 목적과 무관한 상품 광고
금지 행위:
  • 계약 중간에 무관한 팝업 광고
  • 부수 서비스를 필수인 것처럼 표시
  • 주된 상품 가입 완료 전 추가 상품 강요
❌ 위반 사례: 신용카드 발급 중에 "휴대폰 자동이체 가입하면 포인트 추가!"라는 팝업
②
반복간섭
거부 후에도 같은 요구를 반복
금지 행위:
  • "다음에 하기" 선택 후 재팝업
  • 같은 내용 2회 이상 반복 요구
  • 거부 의사 표시 무시하고 재질문
  • 불필요한 재확인 요청
❌ 위반 사례: 마이데이터 "다음에 하기"를 선택해도 "다음에 할까요?"라는 팝업으로 재질문 반복
③
감정적 언어사용
감정을 자극하는 표현으로 압박
금지 행위:
  • 두려움 유발 표현 ("놓치지 마세요")
  • 긴급성 강조 ("지금만", "오늘만")
  • 죄책감 조성 표현
  • 선택을 강요하는 어조
⚠️ 주의: 감정적 표현 자체가 모두 금지되는 것은 아니며, 다른 다크패턴과 결합해 기만성이 커질 때 규제됨
④
감각조작
시각·청각적 자극으로 주의 분산
금지 행위:
  • 깜빡이는 버튼 (시각적 자극)
  • 알림음 (청각적 자극)
  • 색상·크기 과도한 변화
  • 도피 불가능한 자극
⑤
다른 소비자의 활동 알림
거짓 또는 과장된 인기도 표시
금지 행위:
  • "최근 3개월 1000명 가입!" (거짓 통계)
  • "지금 100명이 보고 있습니다" (실시간 조작)
  • 정확하지 않은 인기도 표시
✅ 허용: 정확한 데이터로 다른 소비자의 선택을 알리는 것은 가능 (예: "펀드별 순자산액" 정확히 표시)

✅ 압박형 회피 방법

  • 주된 상품 완료 후에 부수 상품 안내
  • 거부 의사 표시 후 재요구 금지
  • 감정적 표현은 다른 기만과 결합 주의
  • 시각·청각 자극은 도피 가능하게 설계
  • 소비자 활동 정보는 정확하게만 표시
💰 편취유도형 다크패턴 (1개 유형)
금융소비자가 알아채기 어려운 인터페이스 조작으로 비합리적이거나 예상하지 못한 지출을 유도하는 행위
순차공개 가격책정
처음엔 낮은 가격, 마지막에 비용 공개
금지 행위:
  • 첫 화면: 최대 수익률/최소 이율만 표시
  • 계약 진행 중 숨겨진 비용 순차 공개
  • 최종 단계에서 실제 수익률/이율 확인
  • 예상 불가능한 가격 변동
❌ 위반 사례 #1: 대출: "최저 금리 2.5%" (첫 화면) → 신용도 확인 후 "귀고객은 8.5%" (최종 화면) ❌ 위반 사례 #2: 펀드: "평균 수익률 15%" (광고) → 세부 조회: 일부만 15%, 나머지 평균 2~3% (숨겨짐)
금융소비자보호법 제22조
❌ 위험한 설계
단계 1 (광고):
"최저 금리 2%"만 표시
단계 2 (신청):
신용도 확인 화면 (자세히 읽지 않음)
단계 3 (확인):
"귀고객 금리 8.5%" 갑자기 나타남
✅ 올바른 설계
단계 1 (광고):
"금리 2%~8.5% (신용도별)"
우대조건 명확히 표시
단계 2 (신청):
"신용도에 따라 금리가 결정됩니다"
명시적 안내
단계 3 (확인):
"귀고객 금리 8.5%" (예상 범위 내)

✅ 편취유도형 회피 방법

  • 첫 화면에서 예상 가능한 범위 제시
  • 신용도·선택사항 따른 변동을 사전 고지
  • 최종 금리/수익률 확정 전에 예측가능성 높이기
  • 숨겨진 비용이 발생하면 사전 안내 필수
  • 최소~최대 범위를 화면에 명시적으로 표시

다크패턴:
  • 취소 버튼을 3단계 깊숙이 숨김
  • 가입은 1클릭, 취소는 15클릭
  • 취소 확인 전 재가입 제안
결과: 소송 진행 중 (Amazon 항소)

다크패턴:

  • 고객센터 전화 1시간+ 대기
  • 해지 요청 후 계속 청구

CFPB v. One Main Financial (2023)

  • 제외 옵션 찾기 어려움
  • 명확한 비용 공시 없음

📊
FTC 2022년 보고서
"다크패턴 드러내기" (Bringing Dark Patterns to Light)

핵심 발견:
  • 취소 어려움이 가장 흔한 위반
  • 기술적 속임수 광범위 사용
  • 금융·구독 서비스에 가장 많음
권장: 기업 자율규제 + 규제강화

🔍 한국 vs 해외 규제의 차이

  • 한국: 사전 가이드라인 (자율관리 유도) + 금융위 감시
  • 미국: 사후 집행 (소송 + 과징금) + 트렌드 대응
  • EU: 규정 + 기관별 협력 감시
  • 공통: 금융서비스 → 높은 수준의 소비자 보호
✅ 서비스기획자를 위한 실무 체크리스트
온라인 금융상품 판매 과정에서 다크패턴을 회피하고 규제 준수를 위한 단계별 체크리스트입니다.
📋 단계별 체크리스트

1️⃣ 광고 단계

✓
명확하고 정확한 정보 제시
최고 수익률/최저 이율만이 아닌 실제 우대조건·기준 함께 표시
체크: 최저가 아닌 "평균" 수익률도 함께 표시했나?
✓
비용·불이익 균형 공시
혜택(글자 크기/색) vs 손실(글자 크기/색) 균형 맞추기
체크: 수수료는 작은 글자로 표시했나?
✓
거짓/과장 없음
"보장", "필확 수익" 등 단정적 표현 금지
체크: 과장 표현이 있나? 법령 근거가 있나?

2️⃣ 신청 단계

✓
설명 절차 완전성
단계 생략 금지 (제품 이해도 개별 확인)
체크: 투자상품마다 "이해했음" 별도 선택 가능한가?
✓
명확한 질문만
2가지 이상 내용 한꺼번에 묻지 않기
체크: "상품신청 및 마케팅 수신"을 하나로 물었나?
✓
동등한 버튼 디자인
동의·비동의 버튼 크기/색상 동일하게
체크: "신청" 버튼이 "다음에 하기"보다 훨씬 크진 않나?
✓
선택사항 미리선택 금지
모든 옵션 미선택 상태 (필수 아님 한정)
체크: "함께 구매" 옵션이 이미 체크되어 있지 않은가?
✓
클릭 수 최소화
가입 5클릭 이하 + 부수상품은 완료 후 제안
체크: 주상품 신청까지 클릭이 너무 많지 않은가?
✓
중요 정보 가시성
설명서 자동 다운로드, 직관적 배치
체크: 수수료·제약조건을 "자세히 보기"로만 숨기진 않았나?

3️⃣ 거래 유지 단계

✓
추가 개입 최소화
거부 후 반복 요구 금지 ("다음에 할까요?" 팝업 금지)
체크: 사용자가 "다음에 하기" 선택 후 재팝업이 뜨진 않나?
✓
정확한 거래 내용 알림
거짓 인기도나 위급성 표시 금지
체크: "지금 100명이 보고 있습니다" (확인 불가 표시) 제거했나?
✓
비용 변동 사전 고지
신용도·선택사항 따른 수수료/이율 변동 사전 명시
체크: "최종 금리는 신용도에 따라 결정됩니다" 안내했나?

4️⃣ 해지/취소 단계

✓
가입만큼 쉬운 해지
가입 3클릭이면 해지도 3클릭 (또는 더 간단)
체크: 해지 클릭이 가입의 5배 이상은 아닌가?
✓
명확한 해지 경로
"해지" 버튼 위치 명확, 메뉴 깊숙이 숨기지 않기
체크: 해지 버튼이 첫 화면이나 메뉴에 바로 보이는가?
✓
필수 안내 제외 협박 금지
"상담원 연결 필수" 같은 절차는 명확히 안내
체크: 상담원 연결이 필수라면, 온라인 화면에 명시했나?
✓
통합앱 vs 개별 서비스 구분
통합앱 탈퇴 ≠ 개별 금융회사 탈퇴 명확히 구분
체크: 통합앱 해지 시 포함된 개별 회사 탈퇴를 구분했나?
